Section 3

Full Annotated Code

Every line from the notebook, with explanations inline as comments.

Step 1 — Imports

import numpy as np          # for array operations & reshaping
import pandas as pd         # for loading and exploring the CSV

from sklearn.preprocessing import StandardScaler   # normalise features
from sklearn.model_selection import train_test_split # split data
from sklearn import svm                               # SVM classifier
from sklearn.metrics import accuracy_score           # evaluate model

Step 2 — Load & explore data

healthy = pd.read_csv('diabetes.csv')  # load CSV into a DataFrame
# Note: the variable is called 'healthy' but contains both classes

healthy.head()          # view first 5 rows
healthy.shape           # → (768, 9) — 768 rows, 9 columns
healthy.describe()      # statistics: mean, std, min, max for each column

healthy['Outcome'].value_counts()   # 0→500  1→268
# 0 = not diabetic, 1 = diabetic

healthy.groupby('Outcome').mean()
# Key insight: diabetic patients have higher average glucose (~141 vs ~110)
# and higher BMI (~35 vs ~30)

Step 3 — Separate features (X) and label (Y)

X = healthy.drop(columns='Outcome')  # all 8 feature columns
Y = healthy['Outcome']               # the target column (0 or 1)
# X.shape → (768, 8)   Y.shape → (768,)

Step 4 — Standardise (scale) the features

scaler = StandardScaler()   # initialise the scaler

scaler.fit(X)              # learn mean and std from the data
# StandardScaler stores: mean_ and scale_ for each feature

standardized_data = scaler.transform(X)
# Formula applied: z = (x - mean) / std
# Now every feature has mean=0, std=1
# This is critical for SVM — it is distance-based, so scale matters!

X = standardized_data     # overwrite X with the scaled version
Y = healthy['Outcome']   # Y stays unchanged

Step 5 — Train / Test split

X_train, X_test, Y_train, Y_test = train_test_split(
    X, Y,
    test_size=0.2,        # 20% goes to test set  → 154 rows
    stratify=Y,           # preserve class ratio in both splits
    random_state=2        # seed for reproducibility
)
# X.shape=(768,8)  X_train.shape=(614,8)  X_test.shape=(154,8)

Step 6 — Train the SVM classifier

classifier = svm.SVC(kernel='linear')
# SVC = Support Vector Classifier
# kernel='linear' → finds a straight hyperplane to separate classes
# (kernel='rbf' would allow curved boundaries for non-linear data)

classifier.fit(X_train, Y_train)
# The SVM finds the hyperplane with maximum margin
# Only the 'support vectors' (closest points) define it

Step 7 — Evaluate accuracy

X_train_prediction = classifier.predict(X_train)
training_accuracy  = accuracy_score(X_train_prediction, Y_train)
print('Training accuracy:', training_accuracy)   # ~0.786

X_test_prediction = classifier.predict(X_test)
test_accuracy     = accuracy_score(X_test_prediction, Y_test)
print('Test accuracy:', test_accuracy)             # ~0.773

# Small gap between train and test accuracy = good generalisation
# A large gap would indicate overfitting

Step 8 — Predict for a new person

input_data = (5, 166, 72, 19, 175, 25.8, 0.587, 51)
# Pregnancies=5, Glucose=166, BloodPressure=72, SkinThickness=19
# Insulin=175, BMI=25.8, DiabetesPedigree=0.587, Age=51

input_array   = np.asarray(input_data)          # convert to numpy array
input_reshaped = input_array.reshape(1, -1)       # shape: (1, 8) — 1 sample, 8 features
std_data       = scaler.transform(input_reshaped) # scale using SAME scaler (not fit again!)

prediction = classifier.predict(std_data)
print(prediction)     # [1]

if prediction[0] == 0:
    print('The person is NOT diabetic')
else:
    print('The person IS diabetic')   # ← this one prints
Section 4

SVM — How It Works (Visual)

Support Vector Machine finds the hyperplane that separates the two classes with the widest possible margin. The data points on the boundary edges are called support vectors — they are the only points that define the boundary.

The dashed vertical line is the decision boundary (hyperplane). The two faint lines form the margin. Points on the margin edges (ringed circles) are the support vectors. SVM maximises the margin width — wider margin = better generalisation to new data. In 8-dimensional feature space, this "line" becomes an 8D hyperplane.

Section 5

Key Concepts Explained

📐 StandardScaler
Transforms each feature to mean=0, std=1 using:
z = (x − mean) / std

Why? SVM is distance-based. Without scaling, Insulin (0–846) would dominate Pregnancies (0–17) purely due to range, not medical importance.
✂️ train_test_split
Splits data into 80% training + 20% testing. stratify=Y ensures the class ratio (65/35) is preserved in both splits. Without stratify, the test set could accidentally contain too many of one class.
🤖 SVM — kernel='linear'
Finds a flat hyperplane in n-dimensional space separating the two classes with maximum margin. kernel='linear' = straight boundary. kernel='rbf' = curved boundary for non-linearly separable data. Support vectors are the closest points to the boundary.
📊 Accuracy score
Fraction of correctly predicted samples. Training accuracy ≈ 78.6%, Test accuracy ≈ 77.3%. Small gap is healthy. A big gap (high train, low test) = overfitting. On imbalanced data, accuracy alone is misleading — prefer F1-score, AUC-ROC.
⚖️ Class imbalance
500 healthy vs 268 diabetic = imbalanced. A model that always predicts "healthy" gets 65% accuracy without learning anything. That's why both train and test accuracy are checked, and stratify=Y is used.
🔄 reshape(1, -1)
scikit-learn expects input as a 2D array: (n_samples, n_features). A single prediction is shape (8,) — 1D. reshape(1, -1) converts it to (1, 8) — 1 sample, 8 features. Without this, predict() will throw an error.

High glucose (166, above the diabetic threshold of 140) combined with age 51 and moderate family history are the strongest drivers of this prediction. The raw values are first transformed using scaler.transform() — not fit_transform() — so the same scale learned from training data is applied.

Why scaler.transform() and NOT scaler.fit_transform() for new data?

fit_transform() would learn new mean/std from this single input — completely different from what was learned on the 614 training rows. The scaled value would be meaningless. We must use the same scaling parameters that were fit on training data, so scaler.transform() is correct. This principle applies to all preprocessing: fit once on training data, transform everywhere else.

Section 7

Probable Exam / Viva Questions & Answers

Click any question to reveal the answer.

SVM is a distance-based algorithm — it maximises the margin between classes, and margin is measured in terms of Euclidean distance. If features have very different ranges (Insulin: 0–846, Pregnancies: 0–17), the algorithm will assign more importance to Insulin simply because its values are larger, not because it's medically more significant. StandardScaler brings all features to mean=0, std=1, ensuring every feature has equal influence. Without scaling, the SVM boundary would be distorted and accuracy would drop significantly.

Without stratify, the random split might put most diabetic patients into the training set and almost none in the test set (or vice versa). stratify=Y ensures both the training and test sets maintain the same class proportion as the original dataset (≈65% healthy, 35% diabetic). This gives a reliable and fair evaluation of the model's performance on both classes.

Support vectors are the training data points that lie closest to the decision boundary (hyperplane). The SVM optimisation only cares about maximising the margin between these edge points — all other points, being far from the boundary, don't affect where the boundary is placed. If you removed a non-support-vector point from the training set, the model wouldn't change at all. This makes SVM computationally efficient and less sensitive to outliers far from the boundary.

linear kernel: finds a flat (linear) hyperplane to separate the classes. Works well when the data is approximately linearly separable. Simpler, faster, and more interpretable.

RBF (Radial Basis Function) kernel: maps data into a higher-dimensional space and finds a boundary there, which corresponds to a curved boundary in the original space. Works better when classes are not linearly separable. Has a hyperparameter gamma controlling how tightly it wraps around data points.

For this dataset, linear works reasonably well (~77% accuracy). RBF might do slightly better but risks overfitting without cross-validation.

A model that always predicts "not diabetic" would achieve 65% accuracy without learning anything useful. So accuracy alone is deceptive for imbalanced datasets. Better metrics are:
• Precision: of all predicted diabetics, how many actually are?
• Recall (Sensitivity): of all actual diabetics, how many did we catch? (This is critical in medical diagnosis — missing a diabetic is dangerous)
• F1-score: harmonic mean of precision and recall
• AUC-ROC: overall performance across all thresholds

Techniques to handle imbalance: SMOTE (oversample minority), class_weight='balanced' in SVC, or undersample the majority class.

Training accuracy tells you how well the model memorised the training data. Test accuracy tells you how well it generalises to unseen data — this is what matters in the real world.

• Small gap (like in this project, ~1–2%): good generalisation
• High train, low test (e.g. 95% train, 65% test): overfitting — the model memorised noise, not patterns
• Both low: underfitting — the model isn't learning the data at all

Always use an unseen test set to report your final model's performance.

fit_transform() first learns (fits) the mean and std from the data, then transforms it. If you called fit_transform() on the test set or a new input, it would learn completely different statistics from just those few rows, producing meaningless scaled values.

You must use the same mean and std that were learned from the training data. That's what scaler.transform() does — it uses the stored parameters from the earlier scaler.fit(X_train) call. This principle is called preventing data leakage.

train_test_split shuffles the data randomly before splitting. Without a fixed seed, you'd get a different split every run, leading to different results. random_state=2 seeds the random number generator so the shuffle is identical every time — making the experiment reproducible. Any integer works; 2 is just what was chosen here. This is important for debugging, comparison, and reporting results.

scikit-learn's predict() expects input with shape (n_samples, n_features) — a 2D array. When you create a numpy array from a single tuple, it has shape (8,) — a 1D array.

.reshape(1, -1) reshapes it to (1, 8): 1 sample with 8 features. The -1 means "infer this dimension automatically". Without this reshape, sklearn throws a: ValueError: Expected 2D array, got 1D array instead.

It is a genetic risk score calculated from the patient's family history of diabetes. Higher values indicate a stronger hereditary risk. It considers how many first and second-degree relatives had diabetes and how recently. Range is 0.08–2.42. It's one of the more unique features in this dataset — essentially encoding how much your genes predispose you to diabetes.

Several improvements:
1. Handle zero values in Glucose, Insulin, BMI (impute or remove)
2. Use cross-validation (k-fold) instead of a single train/test split for more reliable accuracy estimates
3. Try other kernels (RBF) and tune C and gamma with GridSearchCV
4. Better metrics: report precision, recall, F1, and AUC-ROC — not just accuracy
5. Address class imbalance: use class_weight='balanced' or SMOTE
6. Feature engineering: create interaction terms like Glucose × BMI
7. Compare models: try Random Forest, Logistic Regression, XGBoost and compare performance
